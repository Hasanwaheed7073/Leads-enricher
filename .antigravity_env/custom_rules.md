# Core Directives
1. ZERO-COST ARCHITECTURE: Every library, API, or service used must have a permanent free tier or be open-source.
2. SURGICAL EDITS ONLY: Never rewrite an entire file if only a single function needs changing. Use precise insertions and deletions.
3. SELLABLE QUALITY: All outputs must be production-ready, modular, and packaged so the final product can be deployed and sold as a standalone SaaS.
4. SELF-HEALING: All data pipelines (especially scraping/API calls) must include try/catch blocks with detailed error logging and auto-retry mechanisms.
